# ADR-003: 職業を先に固定せず、カード軸とレリック主導でビルドを成立させる第一案とする

## ステータス
**提案中 (Proposed)**

## コンテキスト (背景)
本プロジェクトでは、グリッド移動、位置取り、状態異常、カード効果、クールタイムなど、すでに複数の戦術要素があります。
この段階で職業を明確に定義すると、職業ごとに初期デッキ、固有パッシブ、成長導線、専用カード群、専用バランス調整などが必要になります。

個人開発では、職業を複数用意した時点で、比較対象として最低でも2〜3職分の完成度が必要になりやすく、実装と調整の負荷が大きくなります。

また、本ゲームでは「回避」「シールド」「状態異常」「位置操作」「準備して吐く大技」など、プレイスタイルの核になりそうな要素がすでに複数あります。
これらは必ずしも職業として先に固定しなくても、カードとレリックの組み合わせから自然にビルドとして成立する可能性があります。

## 決定事項 (Decision)
第一案として、当面は職業を明確に固定しません。
先に、コアとなるカード軸、アイテム、レリック、パッシブを実装し、プレイヤーが組み合わせてビルドを作る方式を採用します。

採用方針は以下の通りです。

* 職業は最初から厳密に分けず、後から整理可能な概念として扱います。
* 先に、プレイスタイルの核になるカード軸を複数用意します。
* レリックやパッシブは、特定の軸を強化する補助要素として扱います。
* 実戦で自然にまとまりが見えた軸だけを、将来的に「職業」として再定義することを検討します。

この方針では、「騎士」「忍者」「ガンナー」のような名前は、最初から職業選択画面で固定するものではなく、ビルド傾向に後から名前を付ける形を優先します。

## カード設計の方針
カードは「腐りにくさ」と「個性」を両立させることを重視します。

各カードは、以下の3要素をなるべく持つ方針とします。

* 単体で最低限機能すること
* 特定条件下でより強くなること
* 他カードや状態異常、地形、レリックと組み合わさるとさらに伸びること

これにより、すべてのカードを量産型の汎用札にせず、それでいて特定状況以外で完全に腐るカードも減らします。

また、条件付きカードを設計する場合でも、条件未達時に完全無価値にならないようにします。

例:

* 低ダメージ + 炎上付与
* 1マス移動 + 小シールド
* 弱攻撃 + 弾丸カード生成
* 位置操作 + 追加条件で高火力

## 先行実装するカード軸の候補
当面は、以下のような汎用性のある複数軸を先に整備する想定です。

* 位置操作軸
  押す、引く、移動する、危険地形へ乗せる
* 防御前進軸
  シールド、被弾前提で前に出る近接戦闘
* 状態異常軸
  炎上、睡眠、弱体、毒などで継続的に有利を作る
* 準備消費軸
  弾丸、チャージ、設置物などを準備し、後で強く使う
* 汎用打点軸
  単体攻撃、直線攻撃、範囲攻撃などの基礎火力

これらは互いに混ぜて使えることを前提とします。
例えば、位置操作 + 炎上、シールド + 近接火力、弾丸 + 状態異常のような組み合わせから、多様なビルドが自然に生まれる状態を目指します。

## ガンナー系ビルドの扱い
ガンナー系のような個性的な軸も、まずは職業ではなくカード群として実装します。

例:

* 近接打撃: 弱い前方攻撃 + 弾丸カード生成
* 装填: 即席の弾丸カードを複数得る
* 弾丸: 炎上弾、氷結弾、貫通弾などの即席カード
* 砲撃: 弾丸を1枚消費して対応した遠距離効果を発動する
* 掃射: 手札の弾丸をまとめて消費して高火力を出す

このような形で、先に「準備して吐く」カード軸として成立させ、十分にまとまりが出た段階で、必要なら将来的にガンナー系職業として整理することを検討します。

## 実装方針メモ
詳細は個別実装時に詰めますが、現時点では以下を重視します。

* 1軸あたり少数のカードから始めます。
* 各カードは単体でも最低限使えるようにします。
* 強力なカードは、数値だけでなく準備、位置、クールタイム、射程などで重さを持たせます。
* 「突破口」になるカードは少数入れますが、万能解答札にはしません。
* 先にカード軸を実戦投入し、手応えがあるものだけを拡張します。

## 帰結 (Consequences)

### メリット (Pros)
* 個人開発で重くなりがちな職業設計を後回しにできます。
* 少数カード単位で実装と検証を進めやすくなります。
* ビルドの多様性を、職業固定ではなく組み合わせから自然に作れます。
* 手応えのある軸だけを後から職業化できるため、無駄な実装を減らせます。
* プレイヤーごとの攻略法や突破口を見つける楽しさを作りやすくなります。

### デメリット / 今後の課題 (Cons / Future Work)
* 職業がない段階では、プレイヤーにビルドの方向性が伝わりにくい可能性があります。
* カード軸が増えすぎると、結果的に職業がないだけで複雑さは高くなるため、追加順序の管理が必要です。
* レリックやパッシブの支援量を誤ると、特定軸だけが過剰に強くなる可能性があります。
* 将来的に本当に職業化するか、最後までビルド制のまま行くかは、プレイ感を見て判断する必要があります。

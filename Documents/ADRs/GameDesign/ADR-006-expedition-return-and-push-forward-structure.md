# ADR-006: RUNは複数遠征で構成し、遠征クリア後に帰還か続行を選ぶ

## ステータス
**ADR-008により一部置き換え (Partially Superseded by ADR-008)**

OverHuntの報酬と帰還時の精算ルールはADR-008を優先する。一般戦闘報酬は遠征帰還時まで保留せず、各戦闘終了時に解決する。ミッションは未決定であり、本ADRの旧案を実装前提にはしない。

## コンテキスト (背景)
一般的なローグライクカードゲームでは、分岐マップ上で次のノードを選びながら進行する形式が多く使われます。
この方式は理解しやすい一方で既視感が強く、本プロジェクトで重視している「固定手札の強化」「MOD報酬」「HP管理によるリスク判断」を十分に前面へ出しづらい面があります。

本ゲームでは、カード枚数を増やすよりも既存カードを育てることを中心にします。
また、ダメージを受けることはHPだけでなく、回復報酬を選ばされることによる強化機会の損失としても機能します。

そのため、進行構造も「どのマスへ進むか」より、「遠征をどこまで続けるか」「安全に帰還するか、追加報酬のために危険を取るか」を中心にします。

## 決定事項 (Decision)
RUNは複数の遠征で構成します。

* 1回のRUNには複数の遠征を設定します。
* 各遠征は、複数の戦闘と遠征ボスで構成します。
* 遠征の基本クリア目標は、その遠征内のボスを倒すことです。
* 遠征ボスを倒した後、プレイヤーは「拠点へ帰還する」か「続けて探索する」かを選びます。
* 拠点へ帰還した場合、OverHuntで得た保留ゴールドを確定します。一般戦闘の報酬とゴールドは、各戦闘終了時にすでに獲得済みです。
* 続けて探索する場合、より強い敵や危険条件付きの戦闘が発生します。
* 続けて探索の報酬はADR-008に従い、保留ゴールドのみとします。
* 続けて探索中に死亡した場合、その時点でRUN失敗とします。

### 現行プロトタイプの遠征構成と引継ぎ

現行の `DefaultRun` は、既存10ステージを `3 / 3 / 3 / 1` ステージの4遠征として構成します。各遠征の最後のステージを遠征ボスとし、ボス撃破後はその遠征に設定されたOverHunt候補へだけ進行できます。最終遠征のボスを倒して街へ帰還した時だけRUN勝利です。

* 遠征1: ステージ1〜3 / OverHunt: ステージ2
* 遠征2: ステージ4〜6 / OverHunt: ステージ5
* 遠征3: ステージ7〜9 / OverHunt: ステージ8
* 遠征4: ステージ10 / OverHunt: ステージ9

街から次遠征へ出発する時、HP、デッキとそこに付与済みのMOD、所持ゴールド、所持アイテムIDは引き継ぎます。アイテムの購入・効果は後続Phaseで追加しますが、所持状態は `RunState` に確保してRUN終了まで再生成しません。OverHuntの保留ゴールドは帰還時に精算済みであり、深度とOverHunt中フラグは次遠征の開始時にリセットします。

## 続けて探索の扱い
続けて探索は、遠征クリア後の任意ハイリスクハイリターン選択として扱います。

* 続けて探索は強制しません。
* プレイヤーは遠征ボス撃破後に、安全に帰還してOverHuntの保留ゴールドを確定できます。
* 続けて探索を選ぶ場合、死亡時RUN失敗であることをUI上で明確に表示します。
* 続けて探索では、敵HP増加、敵攻撃力増加、回復報酬なし、開始マナ減少、危険マス追加などのリスク条件を付与できます。
* 続けて探索の報酬は、ADR-008に従い、街で使う保留ゴールドのみとします。
* 続けて探索に成功した後、さらに続けるか帰還するかを再度選べる設計も許容します。

## ミッションとの関係

ミッションの採用、発令条件、達成報酬、精算タイミングは未決定です。
実装する場合は、先に専用ADRでルールを決めます。

## 報酬方針
遠征クリア時／街への帰還時には、以下を精算します。

* 遠征クリア基本報酬
* OverHuntで得た保留ゴールド
* 今後ミッションを導入した場合のミッション報酬

一般戦闘の三択報酬とゴールドは、ADR-005およびADR-008に従い戦闘終了時に即時解決します。
報酬は、ADR-005の方針に従い、MOD追加を中心にします。
レベルアップは低頻度だが強力な報酬として扱います。
新カード報酬は、遠征ボス後、ショップ、イベントなど特別なタイミングに限定します。

## UI表現
続けて探索の選択肢は、リスクを明確に表示します。

例:

* `街へ帰還する: OverHuntの保留ゴールドを確定`
* `続けて探索する: 強敵戦に挑む / 保留ゴールドを増やす / 死亡時RUN失敗`

この選択はプレイヤーにとって重大なリスク判断なので、曖昧な文言にしません。

## 帰結 (Consequences)

### メリット (Pros)
* STS型の分岐マップに頼らず、独自の進行テンポを作れます。
* 遠征単位で短期目標を作れるため、RUN全体が長くなっても区切りが明確になります。
* 「帰るか、欲張るか」という直感的なリスク判断を毎遠征で作れます。
* HP管理、回復報酬の機会損失、MOD強化が同じ判断軸に集約されます。
* 続けて探索はプレイヤーの任意選択なので、失敗時の納得感を作りやすいです。

### デメリット / 今後の課題 (Cons / Future Work)
* 複数遠征、続けて探索、帰還時の保留ゴールドを管理する状態設計が必要になります。
* 続けて探索の報酬が弱いと選ばれず、強すぎると常に選ぶのが正解になります。
* 遠征クリア後に死亡するとRUN失敗になるため、UIでリスクを明確に伝える必要があります。
* ミッションを導入する場合は、認知負荷と報酬テンポを別途設計する必要があります。
* 帰還後の拠点で何ができるかは、別途設計する必要があります。
